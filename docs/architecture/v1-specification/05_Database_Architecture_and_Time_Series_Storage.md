## 1. Overview

MONEYMAKER uses **TimescaleDB** (PostgreSQL 16 extension) for all persistent storage. TimescaleDB provides transparent time-series optimization through hypertables â€” automatic partitioning, compression, and continuous aggregates.

**Redis 7** serves as a high-speed cache layer for real-time state (latest prices, portfolio state, kill switch flag, signal dedup).

**Schema files**: `infra/docker/init-db/`
- `001_init.sql` â€” Core tables (market data, signals, executions, audit) â€” 161 lines
- `002_ml_tables.sql` â€” ML model tracking and predictions â€” 73 lines
- `003_strategy_tables.sql` â€” Strategy performance and daily summaries â€” 79 lines

**Total**: 8 tables + 1 continuous aggregate, 4 hypertables.

**Extension initialization**: `CREATE EXTENSION IF NOT EXISTS timescaledb;`

## 2. Core Tables

Source: `infra/docker/init-db/001_init.sql`

### ohlcv_bars (Hypertable)

Stores OHLCV candlestick data at all timeframes. Primary data store for market history.

| Column | Type | Constraints | Description |
|---|---|---|---|
| `time` | `TIMESTAMPTZ` | NOT NULL | Bar open timestamp |
| `symbol` | `TEXT` | NOT NULL | Canonical symbol (e.g., `XAU/USD`) |
| `timeframe` | `TEXT` | NOT NULL | M1, M5, M15, H1, H4, D1 |
| `open` | `NUMERIC(20,8)` | NOT NULL | Open price |
| `high` | `NUMERIC(20,8)` | NOT NULL | High price |
| `low` | `NUMERIC(20,8)` | NOT NULL | Low price |
| `close` | `NUMERIC(20,8)` | NOT NULL | Close price |
| `volume` | `NUMERIC(20,8)` | NOT NULL DEFAULT 0 | Volume |
| `tick_count` | `INTEGER` | DEFAULT 0 | Ticks in this bar |
| `spread_avg` | `NUMERIC(10,5)` | DEFAULT 0 | Average spread |
| `source` | `TEXT` | NOT NULL DEFAULT 'unknown' | Data source identifier |

- **Hypertable**: chunk interval 1 day
- **Index**: `idx_ohlcv_symbol_time ON (symbol, timeframe, time DESC)`
- **Compression**: After 7 days, segmentby `symbol,timeframe`

### audit_log (Append-Only)

Immutable audit trail with SHA-256 hash chain.

| Column | Type | Constraints | Description |
|---|---|---|---|
| `id` | `BIGSERIAL` | PRIMARY KEY | Sequence number |
| `created_at` | `TIMESTAMPTZ` | NOT NULL DEFAULT NOW() | Entry timestamp |
| `service` | `TEXT` | NOT NULL | Source service name |
| `action` | `TEXT` | NOT NULL | Action performed |
| `entity_type` | `TEXT` | | Affected entity type |
| `entity_id` | `TEXT` | | Affected entity ID |
| `details` | `JSONB` | NOT NULL DEFAULT '{}' | Action details |
| `prev_hash` | `TEXT` | NOT NULL | Hash of previous entry (GENESIS for first) |
| `hash` | `TEXT` | NOT NULL | SHA-256 hash of this entry |

- **Indexes**: `idx_audit_service_time ON (service, created_at DESC)`, `idx_audit_entity ON (entity_type, entity_id) WHERE entity_type IS NOT NULL`
- **Hash formula**: `SHA256(prev_hash || service || action || details_json || timestamp_iso)`

**Append-only enforcement** via PostgreSQL triggers:

```sql
CREATE OR REPLACE FUNCTION prevent_audit_modification()
RETURNS TRIGGER AS $$
BEGIN
    RAISE EXCEPTION 'Audit log is append-only. Updates and deletes are not permitted.';
    RETURN NULL;
END;
$$ LANGUAGE plpgsql;

-- Two triggers prevent any mutation:
CREATE TRIGGER audit_no_update
    BEFORE UPDATE ON audit_log
    FOR EACH ROW EXECUTE FUNCTION prevent_audit_modification();

CREATE TRIGGER audit_no_delete
    BEFORE DELETE ON audit_log
    FOR EACH ROW EXECUTE FUNCTION prevent_audit_modification();
```

### strategy_performance (Hypertable)

Per-trade results attributed to specific strategies.

| Column | Type | Constraints | Description |
|---|---|---|---|
| `id` | `BIGSERIAL` | PRIMARY KEY | Auto-increment ID |
| `signal_id` | `UUID` | NOT NULL | Originating signal |
| `strategy_name` | `TEXT` | NOT NULL | Strategy that generated signal |
| `symbol` | `TEXT` | NOT NULL | Trading instrument |
| `direction` | `TEXT` | NOT NULL | BUY or SELL |
| `confidence` | `NUMERIC(5,4)` | NOT NULL | Signal confidence |
| `regime` | `TEXT` | | Market regime at signal time |
| `source_tier` | `TEXT` | | TECHNICAL, STATISTICAL_PRIMARY, etc. |
| `entry_price` | `NUMERIC(20,8)` | | Entry price |
| `exit_price` | `NUMERIC(20,8)` | | Exit price (NULL if open) |
| `stop_loss` | `NUMERIC(20,8)` | | Stop-loss level |
| `take_profit` | `NUMERIC(20,8)` | | Take-profit level |
| `profit` | `NUMERIC(20,8)` | | Realized P&L (NULL if still open) |
| `lots` | `NUMERIC(10,4)` | | Position size |
| `status` | `TEXT` | NOT NULL DEFAULT 'OPEN' | OPEN, CLOSED, or REJECTED |
| `opened_at` | `TIMESTAMPTZ` | NOT NULL DEFAULT NOW() | Position open time |
| `closed_at` | `TIMESTAMPTZ` | | Position close time |
| `metadata` | `JSONB` | DEFAULT '{}' | Additional context |

- **Hypertable**: on `opened_at` column, chunk interval 1 day
- **Indexes**: `idx_strategy_perf_name ON (strategy_name, opened_at DESC)`, `idx_strategy_perf_symbol ON (symbol, opened_at DESC)`
- **Compression**: After 30 days, segmentby `strategy_name,symbol`, orderby `opened_at DESC`

### strategy_daily_summary (Continuous Aggregate)

Daily aggregates computed from `strategy_performance`. This is a TimescaleDB **continuous aggregate** (not a simple materialized view), meaning it is incrementally updated.

| Column | Source Expression | Description |
|---|---|---|
| `day` | `time_bucket('1 day', opened_at)` | Trading day |
| `strategy_name` | `strategy_name` | Strategy name |
| `symbol` | `symbol` | Trading instrument |
| `total_signals` | `COUNT(*)` | Total closed trades |
| `wins` | `COUNT(*) FILTER (WHERE profit > 0)` | Winning trades |
| `losses` | `COUNT(*) FILTER (WHERE profit < 0)` | Losing trades |
| `total_profit` | `COALESCE(SUM(profit), 0)` | Net P&L |
| `avg_confidence` | `AVG(confidence)` | Average signal confidence |
| `last_signal_at` | `MAX(opened_at)` | Most recent signal |

- **Filter**: Only `WHERE status = 'CLOSED'` trades
- **Grouped by**: `day, strategy_name, symbol`
- **Refresh policy**: `start_offset = 7 days`, `end_offset = 1 hour`, `schedule_interval = 1 hour`

## 4. TimescaleDB Features

### Compression Policies

| Table | Compress After | Segment By | Order By |
|---|---|---|---|
| `ohlcv_bars` | 7 days | `symbol,timeframe` | `time DESC` (default) |
| `market_ticks` | 1 day | `symbol` | `time DESC` (default) |
| `ml_predictions` | 7 days | `symbol,model_name` | `predicted_at DESC` |
| `strategy_performance` | 30 days | `strategy_name,symbol` | `opened_at DESC` |

Compression ratios typically exceed 90% for time-series data. Segment columns are used for efficient range queries (e.g., filtering by symbol).

### Hypertable Chunking

| Table | Chunk Interval | Rationale |
|---|---|---|
| `ohlcv_bars` | 1 day | ~100K rows/day at 6 timeframes |
| `market_ticks` | 1 hour | High-frequency tick data |
| `ml_predictions` | 1 day (default) | Moderate prediction frequency |
| `strategy_performance` | 1 day (default) | Low-frequency trade data |

## 5. Redis Cache

Redis 7 serves as a high-speed cache layer with AOF persistence and password protection.

| Key Pattern | TTL | Purpose |
|---|---|---|
| `moneymaker:tick:{symbol}` | 300s | Latest tick price (config.yaml `tick_ttl_seconds: 300`) |
| `portfolio:state` | none | Current portfolio positions (persisted by PortfolioStateManager) |
| `kill_switch` | none | Emergency halt flag (`0` or `1`); checked by Algo Engine before every processing cycle |
| `signal:dedup:{signal_id}:{symbol}:{direction}` | 60s | Signal deduplication window (prevents duplicate orders) |

**Connection**: `redis:6379` (Docker), password from `MONEYMAKER_REDIS_PASSWORD` env var.

## 7. Query Patterns

### Latest bars by symbol and timeframe

```sql
SELECT * FROM ohlcv_bars
WHERE symbol = 'XAU/USD' AND timeframe = 'H1'
ORDER BY time DESC
LIMIT 200;
```

### Daily P&L summary

```sql
SELECT * FROM strategy_daily_summary
WHERE day >= NOW() - INTERVAL '30 days'
ORDER BY day DESC;
```

### Strategy performance over last 7 days

```sql
SELECT strategy_name, COUNT(*) AS trades,
       SUM(profit) AS total_pnl,
       AVG(confidence) AS avg_confidence
FROM strategy_performance
WHERE opened_at >= NOW() - INTERVAL '7 days'
  AND status = 'CLOSED'
GROUP BY strategy_name
ORDER BY total_pnl DESC;
```

### Active ML model lookup

```sql
SELECT model_name, version, validation_accuracy, created_at
FROM model_registry
WHERE is_active = TRUE;
```

### Audit trail for a signal

```sql
SELECT * FROM audit_log
WHERE entity_type = 'signal' AND entity_id = '{signal_id}'
ORDER BY created_at ASC;
```

### Verify audit chain integrity

```sql
SELECT id, hash, prev_hash,
       LAG(hash) OVER (ORDER BY id) AS expected_prev_hash,
       CASE WHEN prev_hash = LAG(hash) OVER (ORDER BY id) OR prev_hash = 'GENESIS'
            THEN 'OK' ELSE 'BROKEN' END AS chain_status
FROM audit_log
ORDER BY id;
```

## 8. Data Lifecycle

### Compression Timeline

Data moves through stages based on age:

| Stage | Age | State |
|---|---|---|
| **Hot** | 0 - 1 day | Uncompressed, fastest queries |
| **Warm (ticks)** | > 1 day | Compressed (market_ticks only) |
| **Warm (bars)** | > 7 days | Compressed (ohlcv_bars, ml_predictions) |
| **Cold (strategy)** | > 30 days | Compressed (strategy_performance) |

### Continuous Aggregate Refresh

- **strategy_daily_summary**: Refreshed every 1 hour
  - Looks back 7 days (`start_offset`)
  - Excludes last 1 hour (`end_offset`) to avoid partial-hour data
  - Only includes `CLOSED` trades

### Redis TTLs

| Key Type | TTL | Eviction Behavior |
|---|---|---|
| Tick prices | 300s | Auto-expire; stale data = no cached price |
| Signal dedup | 60s | Auto-expire; allows re-evaluation after window |
| Kill switch | None | Persists until explicitly cleared |
| Portfolio state | None | Persists across service restarts |
